# Unreal 5 ‚Äî Setup Guide

> How to turn this repo skeleton into an openable, playable Unreal project. Unlike a lightweight engine, Unreal's playable third-person base (character, camera, input, sample map) is **binary content** generated by Epic's template ‚Äî it can't be hand-written as text. This guide gets you from skeleton to running.

## What's in the repo already (text scaffold)
- `TheLongNoon.uproject` ‚Äî project definition (UE 5.4, module, plugins: Enhanced Input, Steam, Modeling Tools).
- `Source/` ‚Äî the C++ primary game module (`TheLongNoon`) + build targets. Compiles into a code project.
- `Config/` ‚Äî `DefaultEngine.ini`, `DefaultGame.ini`, `DefaultInput.ini` (Enhanced Input + Steam config).
- `Content/` ‚Äî folder structure only (placeholders); real assets are added in-editor.
- `.gitignore` / `.gitattributes` ‚Äî UE-aware, LFS configured.

## What you add (binary, via the editor)
The third-person character, camera, Enhanced Input assets, and a sample map.

## Option A ‚Äî recommended: start from the Third Person template, then graft our structure
1. Install **Unreal Engine 5.4** via the Epic Games Launcher.
2. Launcher ‚Üí **New Project ‚Üí Games ‚Üí Third Person**, **C++**, target Desktop, Maximum quality. Name it `TheLongNoon`.
3. This generates a working third-person game (character `BP_ThirdPersonCharacter`, Enhanced Input `IMC_Default`/`IA_*`, sample map). Verify it runs (Play).
4. Reconcile with this repo:
   - Copy this repo's `Config/*.ini` additions (Steam, project name) into the generated `Config/`.
   - Move template input assets into `Content/Input`, the character into `Content/Characters/Player`, the map into `Content/Maps` (rename to the Sunhollow greybox).
   - Keep our `Source/` module names (`TheLongNoon`) ‚Äî they match the template's expected names, so C++ lines up.
5. Commit the now-populated `Content/` (LFS on ‚Äî see [version control](version-control-and-branching.md)).

## Option B ‚Äî open this skeleton directly
1. Install UE 5.4.
2. Right-click `TheLongNoon.uproject` ‚Üí **Generate Visual Studio / Xcode project files** (or run UnrealBuildTool).
3. Open the `.uproject`; when prompted to rebuild the missing `TheLongNoon` module, click **Yes** (compiles the C++ module).
4. The project opens with an empty default map. Add a third-person character and Enhanced Input manually (or migrate them from the template) ‚Äî Option A is faster for this.

## First milestone after setup: the Sunhollow greybox
Per the design docs, the vertical slice is **Region 1 ‚Äî The Sunhollow** ([lore](../lore/region-1-the-sunhollow.md), [design](../design/progression-and-gating.md)):
1. Create `Content/Maps/L_Sunhollow_Greybox.umap`.
2. Blockout terrain/props with **Modeling Tools** (enabled).
3. Set a low, warm **DirectionalLight** + **PostProcessVolume** for the golden-hour look ([art guide](../art/style-guide.md)).
4. Drop in the third-person character; set the GameMode default pawn.
5. Author the Enhanced Input actions listed in [project settings](project-settings.md) ¬ß5.
6. Stub the core loop (gather ‚Üí craft ‚Üí prune ‚Üí build) as C++ components + Blueprint test UI.

## Notes
- **Git LFS must be installed before committing `Content/`** (`git lfs install`).
- UE needs a machine with the engine installed to compile/run; CI for UE is heavier than typical (see [version control](version-control-and-branching.md) ¬ß7).
- Engine version is pinned to **5.4** in `.uproject`; if you install a different 5.x, update `EngineAssociation` and re-test.
